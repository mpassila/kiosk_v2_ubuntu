Library of Things (LoT) Locker Kiosk

Functionality & Workflow Guide

Overview

The Library of Things (LoT) Locker Kiosk is a self-service system that allows patrons to check out and return non-traditional library items — such as laptops, projectors, cameras, tools, and other equipment — using secure smart lockers. Unlike the Hold Locker (which is used for patron hold pickups), the LoT system supports a full checkout and return lifecycle with item groups, waivers, condition checks, and permanent locker assignments.

Key Difference from Hold Lockers: In Hold mode, staff pre-loads items for specific patrons to pick up. In LoT mode, items live in the lockers and any eligible patron can check them out, use them, and return them — creating a lending cycle. A smart background process keeps item holds, cancellations, and availability live and up to date.
Configuration: All settings for the LoT Locker Kiosk are managed through the SideEvent SaaS management dashboard.

Key Roles

👤

Staff

Load items into lockers via the add-item wizard. Select item group, door size, and patron eligibility. Inspect returned items for condition. Access the admin view for on-site actions such as configuring item groups, waivers, door sizes, condition check policies, and timers via the SideEvent dashboard.

💻

Patron

Scan their library card, browse available item groups, check out items from lockers, and return them when done.

Item Groups

Items in the LoT system are organized into groups. Each group represents a category of lendable equipment and can have its own settings for door sizes, waivers, images, and condition check rules.

Special Group Types

  • HOLDS — A group that works exactly like the Hold Locker system. Staff pre-loads items for specific patrons, and patrons pick them up by scanning their library card.
  • RETURNS — A dedicated group for processing item returns. When selected, triggers the return workflow instead of checkout.
  • DONATIONS — A group for accepting donated items via the locker system.

Workflow 1: Staff Adds an Item to a Locker

This is the process when a staff member loads a lendable item into the locker system.

1
Staff scans item barcode
System identifies it as an item (not a patron or staff card)
2
Add-Item Wizard opens
Shows the scanned barcode and item title (looked up from the ILS)
3
Staff selects Item Group
Chooses from the available categories (e.g., Laptops, Projectors). If an item type is defined in the ILS integration, the group is detected automatically.
4
Staff selects Door Size
Small, Medium, or Large — based on the physical item size
5
Staff sets Patron Eligibility
"All" — any patron can check out, or "Custom" — reserved for a specific patron ID
6
System assigns next available door
Finds an empty door matching the selected size within the chosen group
7
Locker door opens automatically
Staff places the item inside and closes the door
8
System records the assignment
Item ID, group, door number, patron eligibility, and timestamp are saved to the cloud. The ILS check-in is also performed.

Workflow 2: Patron Checks Out an Item

This is the self-service experience when a patron wants to borrow an item from the locker. Steps 1–3 can be performed in alternative order: the patron can scan their library card first and then select an item group, or select an item group first and then scan their card (the system asks to confirm the selection).

Option A: Scan card first

  1. Patron scans their library card
    Barcode, RFID, or manual entry via on-screen keyboard
  2. System authenticates the patron
    Validates against the ILS. Checks for account blocks.
    • If the patron has existing holds, those are shown first.
  3. Patron selects an item group
    Browses available categories (e.g., Laptops, Cameras).
    • Confirmation is asked to minimise wrong selections.

Option B: Select item first

  1. Patron selects an item group
    Browses available categories displayed as cards on the screen.
  2. Patron scans their library card
    Barcode, RFID, or manual entry via on-screen keyboard.
    • If the patron has existing holds, those are shown first.
  3. System authenticates and confirms selection
    Validates against the ILS. Asks the patron to confirm the selected group.
4
Waiver acceptance (if required)
If the group has a waiver, the patron must read and accept it before proceeding. Approval is saved for records.
5
ILS checkout is processed
The item is checked out to the patron in the library system (Polaris or SIP2)
6
Locker door opens
The assigned door unlocks. System verifies the door actually opened. Bound/paired doors open simultaneously if configured.
7
Patron takes the item and closes the door
A re-open door button is displayed in case the door didn't open or closed too early.
8
System updates the locker status
For normal lockers: the locker is cleared and made available for restocking. For permanent lockers: the locker is marked "Checked Out" and reserved for the item's return.

Permanent Lockers

Some high-value items (e.g., cameras, laptops) can be assigned a permanent locker. The locker stays reserved for that specific item even after checkout. When the patron returns the item, it goes back into the same locker. This ensures consistent tracking and easy staff management of expensive equipment.

Workflow 3: Patron Returns an Item

When a patron brings back a borrowed item, the return process works as follows:

1
Patron scans the item barcode at the kiosk
System recognizes the item as currently checked out
2
System finds an available return locker
Selects an empty door that matches size requirements. For permanent lockers, routes the item back to its assigned door.
3
Locker door opens
Patron places the item inside
4
Condition report (optional)
Patron can report if the item is missing or damaged, and leave a message for staff
5
Patron closes the door
System monitors door status and waits for it to close (30-second timeout)
6
ILS check-in is processed
By default, the item is checked back in through the library system once the door is closed. If delayed check-in is enabled together with enforce staff check, the ILS check-in is deferred until staff completes their condition review, ensuring the item is inspected before being formally returned in the system.
7
System updates locker status
If condition check is enabled, the locker is flagged for staff inspection before the item can be lent again. Otherwise, the item becomes available immediately.
Locker Full on Return: If no empty doors are available for a return, the patron sees a friendly message: "Locker is full. Please return the item to the staff desk." The kiosk will indicate which door sizes (if any) are still available.
Staff-Only Return: If the item type is configured so that patron self-return is not allowed, a "Please return this item to staff" message is shown when the item is scanned. The patron is directed to bring the item to the service desk instead.

Condition Check Workflow

For high-value or sensitive items, the system can enforce a staff condition check after every return. This ensures items are inspected before being made available to the next patron.

1
Patron returns an item
Item is placed in a locker and the door is closed
2
System flags the locker for inspection
The locker is marked with a "Condition Check" flag. The item is NOT available for checkout yet.
3
Staff is notified
The SideEvent dashboard shows which lockers need inspection
4
Staff opens the locker and inspects the item
Verifies the item is complete, undamaged, and ready for the next patron
5
Staff clears the condition check flag
The item becomes available for checkout again
Patron Damage Reports: During return, patrons can self-report missing components or damage using checkboxes and a free-text message field. This information is attached to the transaction and visible to staff.

Waiver System

Each item group can have an optional waiver that patrons must accept before checking out an item. This is useful for equipment that requires acknowledgement of usage policies, liability, or care instructions.

How It Works

  • Waivers are configured per group as HTML content in the SideEvent dashboard.
  • The waiver text can include dynamic variables such as the patron's name, location, and current date.
  • The patron must read and accept the waiver on-screen before the checkout proceeds.
  • Accepted waivers are stored in the cloud (Firestore and Firebase Storage) for record-keeping.
Example Use Case: A library lending laptops may require patrons to accept a technology use policy acknowledging that they are responsible for the equipment and agree to return it by the due date.

Key Features

Group-Based Organization

Items are organized into customizable groups (Laptops, Cameras, etc.) with per-group settings for sizes, waivers, and condition checks.

Permanent Lockers

High-value items can be assigned a permanent locker that stays reserved through the entire checkout-return cycle, ensuring consistent item tracking.

Waiver & Compliance

Per-group waivers ensure patrons acknowledge usage policies. Accepted waivers are stored for legal/compliance records.

Condition Check Enforcement

Returned items can be flagged for staff inspection before being made available again, protecting high-value equipment.

Multi-Language Support

The kiosk supports multiple languages configured per device. A language selector is displayed on the welcome screen.

Accessibility

Font size controls (+/-) on-screen. ADA door preferences supported. On-screen keyboard for manual card entry.

Idle Slideshow

When idle, the kiosk displays a slideshow of item group images. Configurable auto-rotation timer. Any touch or scan wakes the kiosk.

Full ILS Integration

Every checkout and return is recorded in the library's ILS (Polaris or SIP2), keeping circulation data accurate and synchronized.

Door & Locker Management

Door Sizes

Lockers come in configurable sizes (small, medium, large). Each item group can restrict which sizes are available to patrons and staff, ensuring items are placed in appropriately sized lockers.

Door Binding

Some locker configurations use paired doors for oversized items. When a bound door is opened, its partner door opens simultaneously.

Door Failure Detection

After every door open command, the system checks (after a 1.5-second delay) whether the door actually unlocked. Failures are logged and reported to the management dashboard. Faulty doors can be disabled to prevent further assignments.

Door Close Monitoring

During checkout and return, the system actively monitors whether the patron has closed the door. Once the door is closed, the kiosk automatically returns to the home screen after a short delay.

Typical Day-to-Day Operations

Morning Setup

  1. Staff verifies locker status via the SideEvent dashboard or by scanning their staff card at the kiosk.
  2. Staff clears any condition check flags from previously inspected returns.
  3. Staff loads new items by scanning them and using the add-item wizard.

Throughout the Day

  • Patrons arrive, scan their card, select an item category, accept any waivers, and take items from auto-opening lockers.
  • Patrons returning items scan the item barcode, place it in the opened locker, and close the door.
  • Staff monitors the dashboard for condition check alerts and damage reports.
  • Staff restocks returned items (after inspection if required) by clearing the condition check flag.
